# Data Model & Database Schema

This document details the local database architecture, primary table schemas, custom type converters, encrypted preferences, and backup structures of the **Finance Tracker** application.

---

## 🗄️ Database Solution: Room (SQLite)

The app uses **Jetpack Room** as an abstraction layer over native SQLite, delivering reliable, compile-time verified, and reactive database transactions. 

*   **Database Class:** `AppDatabase` (Version: `3`)
*   **Database Name:** `"finance_database"`
*   **Migration Rule:** Destructive fallback (`.fallbackToDestructiveMigration(dropAllTables = true)`), ensuring structural upgrades automatically drop and recreate tables if migrations aren't explicitly declared, preventing crashes on testing devices.

## 🗃️ Room Entity Schemas

### 1. `Category` Entity
Represents the grouping under which expenses and incomes are classified.

*   **Table Name:** `categories`
*   **Fields:**
    *   `id` (`Int`, Primary Key, autogenerated): Unique ID for each category.
    *   `name` (`String`): Descriptive name (e.g., "Food", "Salary").
    *   `type` (`TransactionType`): Determines if this category is valid for `INCOME` or `EXPENSE`.
    *   `iconName` (`String`): Identifier for rendering visual icons dynamically.
    *   `isDefault` (`Boolean`): Set to `true` for system-pre-populated categories. These cannot be deleted.
    *   `isArchived` (`Boolean`): Soft-delete indicator. Archived categories don't show up in add-transaction selectors but historical transactions preserve their relationships.

### 2. `TransactionEntity` Entity
Represents an individual financial record of money received or spent.

*   **Table Name:** `transactions`
*   **Fields:**
    *   `id` (`Int`, Primary Key, autogenerated): Unique ID.
    *   `amount` (`Double`): The currency value of the transaction.
    *   `source` (`String`): Merchant or payer name (e.g., "Starbucks", "Employer").
    *   `date` (`Long`): Epoch timestamp in milliseconds. Used for date range filtering and order sorting.
    *   `categoryId` (`Int`): Foreign relation referencing `categories(id)`.
    *   `type` (`TransactionType`): Redundant check matching whether this is an `INCOME` or an `EXPENSE`.
    *   `notes` (`String`): Personal memos or descriptions.
    *   `paymentMethod` (`String`, default `"Cash"`): Transaction mechanism (e.g., "Cash", "Credit Card", "Debit Card", "Bank Transfer").

## 🔄 Type Converters (`Converters.kt`)
Because SQLite lacks direct support for native Kotlin `enum` fields, Room maps them using:
*   **`TransactionType` Enums:** Persisted in SQLite as string representations (`"INCOME"`, `"EXPENSE"`).
    *   `fromTransactionType(type: TransactionType) -> String`
    *   `toTransactionType(value: String) -> TransactionType`

## 🔒 Encrypted Shared Preferences (`EncryptedPrefsManager`)
For security-critical keys (such as user credentials, encryption keys, or login sessions), storing them in plain XML is vulnerable on rooted devices. The application implements `EncryptedSharedPreferences`:

*   **Engine:** `androidx.security.crypto.EncryptedSharedPreferences`
*   **Encryption Scheme:**
    *   Keys: `AES256_SIV` (Deterministic key encryption)
    *   Values: `AES256_GCM` (Authenticated value encryption)
*   **Fallback Strategy:** In case the device's Android Keystore becomes corrupted or suffers from cryptographic hardware failures, the helper catches the exception and falls back to standard `SharedPreferences` to ensure user settings are preserved and the application doesn't experience catastrophic startup crashes.

## 📦 JSON Data & Backup Structure (`JsonDataManager`)
To support full offline syncs, backups, and restorations, the app encapsulates local tables into a single high-fidelity JSON payload managed via **Moshi** serialization.

### `BackupPayload` Model
Aggregates all critical data for simple transmission or disk saves.
```json
{
  "categories": [
    {
      "id": 1,
      "name": "Food",
      "type": "EXPENSE",
      "iconName": "restaurant",
      "isDefault": true,
      "isArchived": false
    }
  ],
  "transactions": [
    {
      "id": 101,
      "amount": 12.50,
      "source": "Starbucks",
      "date": 1782624220370,
      "categoryId": 1,
      "type": "EXPENSE",
      "notes": "Morning coffee",
      "paymentMethod": "Credit Card"
    }
  ],
  "backupTimestamp": 1782624220370
}
```
*   **`JsonDataManager`** is responsible for reading/writing this layout to local application storage or parsing it during a network import.
